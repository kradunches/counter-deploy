## скалирование приложения и нагрузочное тестирование


был создан файл ./stack.yml где описан deploy приложения

`docker swarm init` - инициализация кластера

`docker stack deploy -c stack.yml counter` - запуск docker swarm

`docker service scale counter_app=4 -d` - можно контролировать количество реплик

В директории `./loadtest` есть файл `counter.js` где выполняется нагрузочное тестирование. Отчёты можно посмотреть в директории `./loadtest/results` в виде html отчётов. Отчёт с подписью polluted запускался без reset, но на результат это не очень сильно повлияло. Но что действительно повлияло - так это запуск тестирования при четырёх репликах. Количество запросов (пропускная способность нашего веб-сервера) увеличилась в 3,14 раза. Поэтому можно сделать вывод, что реплицирование приложения благотворно влияет на производительность, но при этом мы жертвуем простотой настройки.
